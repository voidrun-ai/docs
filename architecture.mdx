## Components

- API server: REST endpoints, SSE streaming, WebSocket upgrades
- MongoDB: stores users, orgs, API keys, sandboxes, and sessions
- Sandbox VM: cloud-hypervisor instance booted from a base image
- Sandbox agent: receives commands, file ops, PTY actions via vsock
- Host networking: Linux bridge and NAT to provide VM connectivity

## Request flow

1. Client calls the REST API with an API key.
2. API server validates the key and loads org context from MongoDB.
3. For sandbox actions, the server reaches the agent over vsock.
4. Output is returned as JSON, SSE events, or WebSocket messages.

## Streaming protocols

- Server-Sent Events (SSE) for exec streams and process attach
- WebSockets for PTY sessions and file watch streams

## Storage paths

- Base images: BASE_IMAGES_DIR
- Instances: INSTANCES_DIR
- Kernel: KERNEL_PATH

See Configuration for defaults and overrides.
