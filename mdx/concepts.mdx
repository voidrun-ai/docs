## Sandbox

A sandbox is a VM created from a base image. It has CPU, memory, disk, and a network identity. Most API calls are scoped to a sandbox ID.

## Image

Images are templates used to create sandboxes. The API provides CRUD operations for image records; the host stores image files in BASE_IMAGES_DIR.

## Sessions

- PTY session: persistent terminal you can reconnect to
- Exec session: a command channel bridged through the agent

## Background processes

Long-running commands can be started and later attached to, waited on, or killed. The API returns a PID for tracking.

## Organizations and API keys

Each API key maps to an organization. Org-scoped endpoints manage keys and provide tenant separation.

## Response envelopes

Most endpoints return a success envelope with status, message, data, and optional meta fields. Errors return an error/details payload.
