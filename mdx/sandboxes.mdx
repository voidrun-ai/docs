---
title: Sandboxes
description: Create, list, inspect, and delete sandbox VMs.
---

All endpoints below require X-API-Key and are prefixed with /api.

## List sandboxes

`GET /api/sandboxes`

Query params:

- page (default 1)
- limit (default from server config; max 100)

## Create sandbox

`POST /api/sandboxes`

```json
{
  "name": "vm-01",
  "templateId": "debian",
  "cpu": 2,
  "mem": 2048,
  "sync": true,
  "envVars": {"DEBUG": "true"}
}
```

Notes:

- name must be DNS-1123 compliant
- cpu between 1 and 8
- mem between 1024 and 16384 MiB
- sync waits briefly for the agent to be ready

## Get sandbox

`GET /api/sandboxes/{id}`

## Delete sandbox

`DELETE /api/sandboxes/{id}`

## Not currently exposed

Handlers exist but routes are commented out in the router:

- POST /api/sandboxes/restore
- POST /api/sandboxes/{id}/stop
- POST /api/sandboxes/{id}/pause
- POST /api/sandboxes/{id}/resume
- POST /api/sandboxes/{id}/snapshot
- GET /api/sandboxes/{id}/snapshots
