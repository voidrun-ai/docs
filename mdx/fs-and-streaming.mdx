This page connects file operations with streaming APIs so you can design end-to-end workflows.

## File operations (REST)

Common endpoints:

- List: GET /api/sandboxes/{id}/files?path=/root
- Upload: POST /api/sandboxes/{id}/files/upload?path=/root/file.bin
- Download: GET /api/sandboxes/{id}/files/download?path=/root/file.bin
- Search: GET /api/sandboxes/{id}/files/search?path=/var/log&pattern=error
- Compress: POST /api/sandboxes/{id}/files/compress?path=/dir&format=tar.gz
- Extract: POST /api/sandboxes/{id}/files/extract?archive=/backup.tar.gz&dest=/restore

See File System for the full list.

## File watch (WebSocket)

Use file watch to stream changes from a sandbox:

1) Start a session with POST /api/sandboxes/{id}/files/watch
2) Connect with GET /api/sandboxes/{id}/files/watch/{sessionId}/stream

This is useful for log tails, build output, and live file sync.

## Exec streaming (SSE)

Use exec-stream for real-time command output:

- POST /api/sandboxes/{id}/exec-stream

The stream emits stdout, stderr, and exit events.

## Session exec stream (NDJSON)

For session-based command execution:

- POST /api/sandboxes/{id}/session-exec-stream

The response is application/x-ndjson with chunked output.

## Background process attach (SSE)

For long-running tasks:

- POST /api/sandboxes/{id}/commands/attach

This lets you reconnect to a PID and keep streaming output.

## End-to-end example

A typical pipeline:

1) Upload input files
2) Run exec-stream to process data
3) Watch /output for new artifacts
4) Download results

Combine these with PTY sessions if you need interactive control.
