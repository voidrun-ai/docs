---
title: Ways to Use Voidrun
description: Common usage patterns and product workflows.
---

Voidrun is a low-level execution platform. Most users build higher-level workflows on top of a few primitives: sandboxes, exec, PTY, files, and streaming.

## AI agent tools

- Per-user or per-agent sandboxes
- Execute tasks with exec or PTY
- Upload inputs, download artifacts
- Stream output to the agent for tool feedback

## CI and build automation

- Create a sandbox per pipeline step
- Run tests or builds with exec-stream
- Persist logs and artifacts via file downloads

## Data analysis and notebooks

- Upload datasets to the sandbox
- Execute analysis scripts
- Stream outputs and charts
n
## Secure code execution

- Run untrusted or user-provided code in an isolated VM
- Enforce timeouts and resource constraints
n
## Developer environments

- Provide ephemeral VMs with a consistent base image
- Use PTY sessions for interactive work
- Capture files or snapshots for later review

If you want SDK-driven workflows, start with the TypeScript SDK page. For REST-first usage, see the API Guide pages under Agent Tools.
