---
title: Organizations and API Keys
description: Organization lookup and API key management.
---

All endpoints below require X-API-Key and are prefixed with /api.

## Get current organization

`GET /api/orgs/me`

Response includes the org id and name associated with the API key.

## List API keys

`GET /api/orgs/{orgId}/apikeys`

## Create API key

`POST /api/orgs/{orgId}/apikeys`

```json
{
  "keyName": "CI Key"
}
```

## Revoke API key

`DELETE /api/orgs/{orgId}/apikeys/{keyId}`

## Activate or deactivate API key

`POST /api/orgs/{orgId}/apikeys/{keyId}/activate`

```json
{
  "isActive": true
}
```

## Touch API key (update last-used)

`PATCH /api/orgs/{orgId}/apikeys/{keyId}/touch`

Note: orgId must match the organization associated with the API key.
