---
title: File System
description: File upload, download, search, compress, and watch operations.
---

All endpoints below require X-API-Key and are prefixed with /api.

## List files

`GET /api/sandboxes/{id}/files?path=/root`

## Download file

`GET /api/sandboxes/{id}/files/download?path=/root/output.log`

Returns application/octet-stream.

## Upload file

`POST /api/sandboxes/{id}/files/upload?path=/root/target.bin`

Two modes:

- Multipart form upload (field name file, max 5 MB)
- Raw binary upload (Content-Type: application/octet-stream)

## Create directory

`POST /api/sandboxes/{id}/files/mkdir?path=/new/dir`

## Create file

`POST /api/sandboxes/{id}/files/create?path=/file.txt`

## Copy file or directory

`POST /api/sandboxes/{id}/files/copy?from=/src&to=/dst`

## Move file or directory

`POST /api/sandboxes/{id}/files/move?from=/src&to=/dst`

## Delete file or directory

`DELETE /api/sandboxes/{id}/files?path=/root/file.txt`

## Stat file

`GET /api/sandboxes/{id}/files/stat?path=/root/file.txt`

## Read head or tail

`GET /api/sandboxes/{id}/files/head-tail?path=/var/log/syslog&lines=50&head=true`

## Change permissions

`POST /api/sandboxes/{id}/files/chmod?path=/script.sh&mode=755`

## Disk usage

`GET /api/sandboxes/{id}/files/du?path=/dir`

## Search files

`GET /api/sandboxes/{id}/files/search?path=/var/log&pattern=error`

## Compress

`POST /api/sandboxes/{id}/files/compress?path=/dir&format=tar.gz`

Supported formats: tar, tar.gz, zip.

## Extract

`POST /api/sandboxes/{id}/files/extract?archive=/backup.tar.gz&dest=/restore`

Supported: .tar, .tar.gz, .tgz, .zip.

## Watch files

Start a watch session:

`POST /api/sandboxes/{id}/files/watch`

```json
{
  "path": "/app/logs",
  "recursive": true,
  "ignoreHidden": true
}
```

Stream events (WebSocket):

`GET /api/sandboxes/{id}/files/watch/{sessionId}/stream`
