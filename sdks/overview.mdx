# SDKs & API

This page is the **integration hub** for VoidRun: packages to install, how to authenticate, how SDKs relate to the HTTP API, and where to go next. Use it together with the [Quickstart](/getting-started/quickstart) and [Code execution](/core-concepts/code-execution) guides.

## Official SDKs

| Runtime | Package | Install |
|--------|---------|---------|
| Node.js (ESM + TypeScript) | [`@voidrun/sdk`](https://www.npmjs.com/package/@voidrun/sdk) | `npm install @voidrun/sdk` |
| Python 3.9+ | [`voidrun`](https://pypi.org/project/voidrun/) | `pip install voidrun` |

Both SDKs are generated from and aligned with the same **OpenAPI** description as the VoidRun service. For low-level request/response shapes, use the generated types in each package or the OpenAPI document shipped with the server.

## Authentication

Authenticated endpoints expect your API key on every request:

```http
X-API-Key: <your-api-key>
```

Use this header for direct `curl` or custom HTTP clients. The official SDKs attach it for you when you configure an API key.

<Warning>
Do not assume `Authorization: Bearer â€¦` unless your operator explicitly documents it. The public OpenAPI spec describes **ApiKey** auth via **`X-API-Key`**.
</Warning>

### Environment variables

Typical configuration for the **hosted** VoidRun platform:

| Variable | Purpose |
|----------|---------|
| `VR_API_KEY` or `API_KEY` | API key (Python accepts either; TypeScript reads `VR_API_KEY` when constructing `VoidRun`). |
| `VR_ORG_ID` | Optional default organization id (Python) when your setup requires it. |

Both SDKs use the **same packaged default API base URL** as OpenAPI **`servers`** (see **API Reference**) when you do not override it.

**Self-hosted:** set **`VR_API_URL`** or **`API_URL`**, or pass **`baseUrl` / `base_url=`**, to your deploymentâ€™s API root. See [Self-hosted deployments](/advanced/self-hosted).

## Base URL and paths

- The **base URL** is the prefix for all REST routes (no trailing slash issues: follow the SDK README).
- Path segments (e.g. `/sandboxes`, `/sandboxes/{id}/exec`) are defined in OpenAPI. Older examples sometimes used `/v1` or a different host; **confirm the base URL in your dashboard or deployment docs**.

## REST API overview

- **Contract:** OpenAPI 3 describes sandboxes, execution, filesystem, PTY, background commands, watch endpoints, and error models.
- **Success payloads** often use a wrapper with a nested `data` object; SDKs may unwrap one level for ergonomics (see below).
- **Errors:** JSON bodies commonly include **`error`** and **`details`**. Log both when debugging; SDKs increasingly concatenate them into thrown error messages.

### Quick comparison

| Topic | TypeScript | Python |
|-------|------------|--------|
| Package | [`@voidrun/sdk`](https://www.npmjs.com/package/@voidrun/sdk) | [`voidrun`](https://pypi.org/project/voidrun/) |
| Base URL if env unset | Packaged default (`BASE_PATH`) | Packaged default (`DEFAULT_API_BASE_URL`, same host) |
| Shell exec result | `result.data?.stdout` (under `ExecResponse`) | `result.data.stdout` (inner payload on `VoidRunResponse`) |
| Interpreter result | `result.stdout` on `CodeExecutionResult` | `result.stdout` on `CodeExecutionResult` |
| Async | `async`/`await` throughout | `AsyncVoidRun` + `*_async` where applicable |
