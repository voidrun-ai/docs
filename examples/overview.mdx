# Examples

You use these guides when you already know [how to authenticate](/sdks/overview) and want **copy-adaptable patterns** for real products. Each example lives on its own page so you can follow it end to end without scrolling through unrelated scenarios.

## What you need first

Before you run the code on any example page, you complete this checklist:

1. You create an API key in the [dashboard](https://dash.void-run.com) (or your operator issues one).
2. You set **`VR_API_KEY`** in your environment.
3. On the **hosted** platform you rely on the SDK default API host (same as OpenAPI **`servers`**). For **self-hosted** deployments you set **`VR_API_URL`** / **`API_URL`** or **`baseUrl` / `base_url=`**: see [Self-hosted deployments](/advanced/self-hosted).
4. You install **`@voidrun/sdk`** or **`voidrun`** as described in the [Quickstart](/getting-started/quickstart).

<Note>
On the hosted VoidRun platform, valid **system images** are **`code`**, **`max`**, and **`docker`** (see [System images](/environments/custom-images#system-images)). You confirm names and versions with **`GET /images`** or the dashboard. Self-hosted deployments may differ.
</Note>
